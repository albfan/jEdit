Mac OS Plugin Version 2.0.2 (Oct. 23, 2002)
By Kris Kopicki <kris_k@iprimus.com.au>

Provides Mac OS specific features, such as:

  • jEdit exits cleanly when using Command-Q or "Quit" from the "File" menu.
  • Files created or associated with jEdit can now be opened from the Finder, by either double-clicking the file's icon or dragging the file's icon onto the jEdit application icon.
  • jEdit preserves existing type/creator codes and is assigned as the creator of new files (Default). Can also be set to ignore existing type/creator codes.
  • "Preferences..." in the application menu can be used to access jEdit's Global Options.
  • "About jEdit..." in the application menu can be used to access the About Dialog.
  • The menu's can be placed in the Mac OS X menu bar. This is not recommended, as there are problems with Java on Mac OS X.
  • You can enable live resizing of windows.
  • You can show current and recent buffers in the finder.
  • You can run AppleScripts (compiled, uncompiled and standalone). Your scripts must be located in the macros folder in the jEdit folder. They can be either plain text or compiled scripts. Scripts must have their type and creator set correctly, have a .scpt (for compiled scripts) or .applescript (for uncompiled scripts) extension, or both.
  • The text area can be set to use the system default for highlighting text.

Known Issues:

  • Files opened in the Finder before jEdit is running are not opened in jEdit (10.0.x - 10.1.x).
  • Scripts that require user interaction are not supported.
