Mac OS Plugin Version 1.2 (Apr. 20, 2002)
By Kris Kopicki <krisk4@mac.com>

Provides Mac OS specific features, such as:

  • jEdit exits cleanly when using Command-Q or "Quit" from the "File" menu.
  • Files created or associated with jEdit can now be opened from the Finder, by either double-clicking the file's icon or dragging the file's icon onto the jEdit application icon.
  • jEdit preserves existing type/creator codes and is assigned as the creator of new files (Default). Can also be set to ingore existing type/creator codes.
  • "Preferences..." in the application menu can be used to access jEdit's Global Options.
  • "About jEdit..." in the application menu can be used to access the About Dialog.
  • The menu's can be placed in the Mac OS X menu bar. This is not recommended, as there are problems with Java on Mac OS X.

Known Issues:

  • "Preferences..." in the application menu will only work under Mac OS X 10.1.3 or later. If using 10.1.3, the "Java 1.3.1 update 1" must be installed.
  • Files opened in the Finder before jEdit is running are not opened in jEdit (Apple Java bug).
